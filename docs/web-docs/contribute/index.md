Contributions are welcome across the project: new gears, bug fixes, documentation, testing, performance, and developer experience. This section is the public contributor subset; the canonical quickstart lives in the repository [CONTRIBUTING.md](https://github.com/constructorfabric/gears-rust/blob/main/CONTRIBUTING.md).

- **[Development setup](./development-setup/)** — prerequisites, cloning with submodules, and building.
- **[Code contribution guide](./code-contribution-guide/)** — branch, commit (with DCO sign-off), PR, and review workflow.
- **[Spec-driven workflow (SDD + Studio)](./spec-driven-workflow/)** — the specification templates and Constructor Studio.
- **[Add or change a gear](./add-or-change-a-gear/)** — conventions specific to gears.
- **[Architecture and quality gates](./architecture-and-quality-gates/)** — the lints, checks, and CI that must pass.
- **[Release process](./release-process/)** — SemVer rules and release automation.

## Contribute to the docs

- **[Documentation contribution](./documentation/)** — how to edit and preview these docs.
- **[Translation guide](./translation-guide/)** — localizing pages.

## Getting help

- **GitHub Issues** — bug reports and feature requests.
- **GitHub Discussions** — questions and general discussion.
- **Code examples** — look at existing gears for patterns.

By contributing you agree to follow the project's Code of Conduct.
